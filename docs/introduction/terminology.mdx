# Terminology

<table data-full-width="false">
  <thead>
    <tr>
      <th width="191">Terms</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <a href="../working-with-workspaces/">Workspace</a>
      </td>
      <td>
        A collaborative environment for organizing and managing applications and
        workflows.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-applications/">Application</a>
      </td>
      <td>
        A container within a workspace that holds workflows, resources, and
        schemas.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/">Workflow</a>
      </td>
      <td>
        A sequence of steps designed to complete a process or achieve a business
        goal.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-activities/">Activity</a>
      </td>
      <td>
        A fundamental task or operation within a workflow, like data entry or
        API requests.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-applications/variables">Variable</a>
      </td>
      <td>
        Configurable parameters used for dynamic data storage and workflow
        customization.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/#id-1.-create-a-workflow">Trigger</a>
      </td>
      <td>
        An event or condition that initiates a workflow, such as time-based or
        user actions.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-applications/schemas">Schema</a>
      </td>
      <td>
        Defines the data structure, including fields and data types, used in
        workflows.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-applications/resources">Resource</a>
      </td>
      <td>
        External configurations or connections (e.g., Auth, ABS) used in
        workflows.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/creating-and-using-a-subflow.mdx">
          Subflow
        </a>
      </td>
      <td>
        A reusable sequence of steps within a larger workflow to modularize
        complex tasks.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/mapping/">
          Mapper
        </a>
      </td>
      <td>
        Transforms and maps data between different formats or structures within
        workflows.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/mapping/allof-and-anyof">
          Mapper Function
        </a>
      </td>
      <td>
        Performs data transformations using built-in functions for complex data
        mappings.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-activities/#webhook">Webhook</a>
      </td>
      <td>
        Enables real-time communication by sending HTTP requests when specific
        events occur.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-activities/#generateerror">Error Handling</a>
      </td>
      <td>
        Manages and responds to errors within workflows for robust and resilient
        execution.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workspaces/managing-workspace-level-entities/managing-lookup-values/">
          Lookup Table
        </a>
      </td>
      <td>
        A key-value data structure for efficient data retrieval and mapping in
        workflows.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/transition-lines">
          Transition Line
        </a>
      </td>
      <td>
        Connects activities in a workflow, with optional conditions to control
        flow logic.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../api-manager/">API Manager</a>
      </td>
      <td>
        Manages API collections, access profiles, and secure API tokens for
        integrations.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workspaces/managing-workspace-level-entities/managing-secrets/">
          Secret Manager
        </a>
      </td>
      <td>
        Secures sensitive information like API keys, supporting
        environment-specific storage.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/publishing-workflows">
          Publish
        </a>
      </td>
      <td>
        Publish a specific version of the workflow, making it ready for
        deployment.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/publishing-workflows">
          Deploy
        </a>
      </td>
      <td>
        Moves the published version to the Development environment for
        execution.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/publishing-workflows">
          Promote
        </a>
      </td>
      <td>
        Advances the deployed version to Production environments with approvals.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-workflows/managing-workflows/testing-a-workflow">
          Test
        </a>
      </td>
      <td>
        Verifies the workflow functionality by simulating input data and
        checking output.
      </td>
    </tr>
    <tr>
      <td>
        <a href="../working-with-the-schema-registry/">Schema Registry</a>
      </td>
      <td>
        A centralized location for managing, versioning, and sharing schema
        definitions.
      </td>
    </tr>
  </tbody>
</table>
